# AI Office — правила и процесс

Здесь лежит **единая структура** для работы агентов в этом репозитории.

## Как устроены файлы

| Тип | Назначение |
|-----|------------|
| **`.mdc`** | Жёсткие инструкции для агента (роль, запреты, обязательные действия). Подхватываются Cursor из `.cursor/rules/`. |
| **`.md`** | Описание процесса, шаблоны, справка, примеры (без «закона» для агента). |

Симлинк (рекомендуется): `.cursor/rules/ai-office` → каталог `ai-office/` в **корне репозитория** — чтобы Cursor подхватывал все `.mdc`. Путь относительный зависит от расположения `.cursor/` (часто `../../ai-office`).

## Навигация

Ссылки ниже — как в репозитории: от корня **`ai-office/`** (для кликабельности относительные пути в скобках).

- **Вход задач:** [`ai-office/task-intake.md`](task-intake.md) (шаблон постановки).
- **Глобальные правила агента:** [`ai-office/global/`](global/) — стандарты, контракты передачи, безопасность.
- **Команды (доменные):** [`ai-office/commands/`](commands/) — например `create_new_ver` для `mt5-bot/`.
- **Роли:** [`ai-office/agents/`](agents/) — Team Lead, Programmer, Tester, Quant, Infosec (Ися).
- **Процесс:** [`ai-office/process/`](process/) — цепочка, доработки.
- **Шаблоны:** [`ai-office/templates/`](templates/) — задача, handoff, отчёты.
- **Чек-листы:** [`ai-office/checklists/`](checklists/) — ревью, QA, релиз.
- **Политики:** [`ai-office/policies/`](policies/) — ветки/PR/MR, approve и выкладка.

## Принцип маршрутизации

**Весь рабочий вход по реализации — только через Team Lead (Туся).**  
Исключения и особые случаи зафиксированы в `ai-office/global/handoff-contracts.mdc` и в ролях агентов.

**User-facing:** **Team Lead** — основной вход для реализации, merge/release и итогов по маршрутизируемой аналитике; **Programmer**, **Tester**, **Quant** — внутренние роли. **Infosec (Ися)** — см. **§6** в `ai-office/global/handoff-contracts.mdc` (двойной вход: через Тимлида или прямой вызов агента для security review).

## Связь с `docs/`

Исторические копии шаблонов могут оставаться в `docs/templates/`; **канон для офиса** — эта папка `ai-office/`. При расхождении ориентируйся на `ai-office/`.
