# Режим доработки (rework loop)

Когда тестировщик **не** подписывает релиз кандидата, цепочка возвращается к разработчику.

## Триггер

- Любой пункт `ai-office/checklists/qa-checklist.md` не выполнен.
- Не выполнены Acceptance Criteria из постановки.

## Шаги

1. **Труся** фиксирует результат в `ai-office/templates/tester-report-template.md` (статус: **FAIL**): шаги, ожидание, факт, логи/скриншоты.
2. **Пуся** устраняет замечания в рамках той же Task/Story (или эскалирует неясность **Тусе**).
3. **Пуся** снова передаёт работу **Трусе** через `ai-office/templates/programmer-handoff-template.md` (указать новый коммит/изменения).
4. **Труся** повторяет полный QA прогон (или согласованный с Тусей subset для минорного фикса — по умолчанию полный).
5. Только после **PASS** — передача **Тусе** на финальное ревью.

## Запрещено

- «Тихо» передавать правки пользователю без Труси/Туси в стандартном режиме.
- Смешивать разные Task в одном handoff без явной пометки в шаблоне.
